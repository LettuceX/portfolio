Customer Service Portal & ERP Integration

Role: Backend Developer Team: 5 People
Architecture: Middleware Layer, RESTful API, Legacy Integration
Key Technologies: NetSuite API (SuiteTalk), SQL, React, C#/Node.js

Project Description

Developed a client-facing web portal to modernize the company's sales and support workflow. This platform allows the customers to configure products, track orders in real-time, and manage support tickets.

1. ERP Integration & Middleware

Bridged a modern web application with a monolithic internal ERP (NetSuite) to provide a "Single Source of Truth."

  • API Consumption: Built a middleware layer to consume NetSuite APIs, pulling complex product catalogs, pricing models, and inventory data to display on the customer-facing web portal.
  • Interoperability: Designed data mapping strategies to resolve schema mismatches between the rigid legacy ERP structure and the flexible modern web database.

2. Data Synchronization & Consistency

Implemented robust synchronization strategies to ensure customers see exactly what the sales team sees.

  • Caching Strategy: Developed a local caching system to store read-only copies of order history, eliminating latency from slow ERP queries and ensuring instant page loads for users.
  • Asynchronous Updates: Utilized background workers to poll for status changes (e.g., "Shipped"), ensuring the portal reflects the live status of physical goods without blocking the user interface.
